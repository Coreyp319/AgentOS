⚠Remote access exposes unauthenticated desktop UIs to your whole tailnet. Read this before you copy anything.

These UIs have no per-request login — tailnet membership is the entire authorization boundary. Anyone on your Tailscale account (every device you've added or shared, and your account if it's compromised) can reach, with no password:

  • :8765Lucid — generate explicit (18+) content and drive the GPU / VRAM lease
  • :9123 / :9124Status & Models panels — your system state and model library
  • :9119Hermes board — can drive the agent
  • :8770Share hub — push photos straight into your box
Your credentials stay on-box

This installer (:9125) is never put on the tailnet — your Hugging Face / Civitai tokens live in your OS keyring (or a disclosed 0600 file) and never leave this machine. The Hermes control API (:8642) and ComfyUI (:8188) are deliberately not exposed — keep them that way.

serve, never funnel

tailscale serve = tailnet-only. tailscale funnel = the public internet — never run it. Verify every line of tailscale serve status reads “tailnet only.” Never port-forward, bind 0.0.0.0, or use funnel/ngrok.

Reboot gap — eyes open

Leaving tailscaled enabled keeps these unauthenticated UIs reachable across every reboot — only do that with 2FA + a tag:agentos ACL in place. Disabled, your box drops off the tailnet after a reboot until you re-run sudo systemctl enable --now tailscaled.

Lower the blast radius

Turn on Tailscale 2FA (highest leverage), apply a tag:agentos ACL so only your devices reach these ports, and never set GATEWAY_ALLOW_ALL_USERS=true.

Bottom line — use serve (never funnel), keep 2FA on, and restrict to your own devices with a tag:agentos ACL.

Run these yourself — the wizard never runs them (they need sudo, and it deliberately can't expose itself):

sudo pacman -S tailscale
sudo systemctl enable --now tailscaled      # persist this — closes the reboot gap (only with 2FA + tag:agentos ACL)
sudo tailscale up                           # opens an auth URL; use the same account on your phone
integrations/agentosd-remote.sh up          # serve the UIs (down / status to reverse / inspect)

See docs/REMOTE-ACCESS.md for the tag:agentos ACL JSON + hardening.
